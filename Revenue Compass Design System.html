01 — Brand motif

A compass that breathes.

The brandmark is a 24×24 compass dial — circle, four cardinal ticks, and an asymmetric needle filled at 18% opacity. The motif recurs as a hairline pattern in empty states, as a glow-haloed status pip on the topbar, and as the silent watermark on tier-1 dashboard panels. It is never animated as a logo. It is allowed to pulse only as a status indicator.

02 — Color

Graphite, with a single signal.

Two scales do almost all the work. Graphite handles every surface, border, and text rank. Signal — an electric chartreuse — is reserved for primary action, focus, status, and one chart series at a time. State colours (positive / warning / critical) only ever appear inside pills, alerts, and chart fills.

04 — Geometry

Spacing. Radius. Elevation.

05 — Iconography

Drawn on a 24 grid. Stroked at 1.5.

A proprietary set drawn in-house. 24×24 viewbox, 1.5-px stroke, square caps with optical rounding only on extremities, 2-px corner radius. Subtle asymmetries are intentional — you'll notice that the compass needle, the spark, the lightning, and the trend marker all skew slightly off-grid. That asymmetry is the system's signature.

07 — Empty · Loading

Patient. Specific. Never cute.

Empty states use the dial motif at low opacity, a single sentence of guidance, and one CTA — never zero, never two. Skeletons shimmer at 1.6s, only ever in graphite-700 → 600 — the accent is forbidden in placeholders.

08 — Motion

Five durations. One curve does the work.

Standard motion uses cubic-bezier(.2,.7,0,1) — a long, late-settling decel that reads as authoritative. Emphasis curve (.16,1,.3,1) is reserved for hero entries; exit curve is symmetric. Below: the timing scale.

// Framer Motion — system defaults const ease = [0.2, 0.7, 0, 1]; const easeEmphasis = [0.16, 1, 0.3, 1]; export const rcSpring = { type: "tween", ease, duration: 0.24 }; export const rcEntry = { initial: { y: 6, opacity: 0, filter: "blur(4px)" }, animate: { y: 0, opacity: 1, filter: "blur(0)" }, transition: { ...rcSpring, duration: 0.48, ease: easeEmphasis } };
// Animation principles // 1 · only one element moves at a time // — siblings stagger, never collide // 2 · stagger 40ms between peer elements // 3 · enter ≠ exit duration; exit is faster // (180ms · ease-exit) // 4 · accent glow scales 1→1.04 max, // never bigger — the value, not the chrome, // is what the eye is supposed to follow // 5 · charts grow once on mount, never on // filter change — re-renders cross-fade

This document is the canonical specification for Revenue Compass UI. Diverge from it only with a written exception filed in /design-system/exceptions/. Tokens are enforced via lint; icons are enforced via codegen. Components ship with visual regression tests at three viewport widths.

Avoid: generic AI gradients · neon cyberpunk · excessive glassmorphism · cartoonish onboarding · flat startup defaults · "friendly AI" rounding · stock Tailwind chrome.
